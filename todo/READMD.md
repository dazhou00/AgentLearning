本 ToDoList 完全贴合之前的学习路线设计，适配在职前端开发者的工作节奏，标准周期 8 个月（32 周，含 3 周弹性缓冲期），每周固定投入 15-20 小时（工作日每日 2-3 小时，周末 5-8 小时），所有任务标注明确优先级、可执行动作、验收标准，无需二次拆解，直接跟着执行即可。

# 前置说明

# 优先级规则

| 优先级 | 定义       | 执行要求                                       |
| ------ | ---------- | ---------------------------------------------- |
| P0     | 核心必做项 | 100% 完成，未完成不得进入下一阶段 / 下一周     |
| P1     | 能力提升项 | 建议完成，贴合企业级开发需求，完成度不低于 80% |
| P2     | 拓展选做项 | 有余力再做，拓宽技术边界，不影响主线进度       |

# 时间分配规则

- 工作日：优先完成理论学习、小 Demo 手写、API 对接、基础代码封装等碎片化任务，每日固定 2-3 小时
- 周末：集中完成项目实战、全流程开发、项目部署、阶段复盘等需要整块时间的任务
- 弹性缓冲：每 2 周预留 1 个周末的缓冲时间，应对工作加班、突发事项，避免进度断层

# 核心执行原则

1. 严格按阶段推进，完成当前周的验收 Checklist 再进入下一周，坚决不跳步
2. 优先吃透 JS/TS 生态技术栈，Python 相关内容均为 P2 选做，不盲目切换技术栈
3. 每学一个知识点必须手写对应 Demo，拒绝 “只看不学、只调 API 不理解原理”
4. 每阶段结束必须做复盘，补全薄弱项，再进入下一阶段
